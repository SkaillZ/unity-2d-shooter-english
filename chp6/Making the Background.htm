Chapter 6. Making the Background

In the last chapter, we used layers for collision detection and animation events.

Now, we’re going to make a background for the game screen.

In a nutshell, this is the process for making the background:

 1:  Make 3 separate images for the background.
 2:  Make the background scroll.
 3:  Each image will individually scroll at different speeds (to simulate parallax)

6.1. Displaying the Three Images

For our background, we’ll apply a texture to a Quad.

First, make an empty game object, and name it Backgrounds.

6.2. Make the Front

Next, make a Quad by selecting the “GameObject -> Create -> Quad” menu item. Name it Front, and make it a child of Backgrounds.

Set the Transform of both Backgrounds and Front to be X 0 Y 0 Z 0.

Let’s change the size of Front. Set its Scale to X 8 Y 6 Z 1.

From the Materials folder, drag and drop Background-Front onto the Quad.

After dragging, does the background fill the screen, like in Figure 6.1?

Figure 6.1:

If Your Image Does Not Fill The Screen

It’s possible you forgot to set up the Main Camera.

Check to make sure your 、Projection and Size settings match those in the image.

6.3. Make the Middle and Back

Duplicate Front, and name the objects Middle and Back.

Next, change the materials for Middle and Back. Drag the corresponding materials onto the appropriate items in the Hierarchy view.

6.4. Adjust the Three Images’ Positions

Set the Front Z position to 0.1, the Middle Z position to 0.2, and the Back Z position to 0.3.

Now each image should display correctly.

6.5. Making the Background Scroll

Create Background.cs in the Scripts folder.

Background.cs

using UnityEngine;

public class Background : MonoBehaviour
{
        // Scrolling Speed
        public float speed = 0.1f;

        void Update ()
        {
                // Y will go from 0 to 1 as time goes on. When the Y value is 1, it will return to 0.
                float y = Mathf.Repeat (Time.time * speed, 1);

                // Make an offset value for the Y value
                Vector2 offset = new Vector2 (0, y);

                // Set the material offset.
                renderer.sharedMaterial.SetTextureOffset ("_MainTex", offset);
        }
}

Attach Background.cs to each of Front, Middle and Back.

Next, to make the texture scrollable, make sure to set the Texture’s Wrap Mode to Repeat.

Play the game to check your work. The background should scroll.

Let's try and change the scroll speed of each layer.

Set the Front’s Speed to 0.1, the Middle’s Speed to 0.08, and the Back’s Speed to 0.02.

Figure 6.2: Front

Figure 6.3: Middle

Figure 6.4: Back

Start the game and see what happens. Each background layer should scroll at different rates.

Sprites Cannot Change Their Offset

There’s a reason we didn’t use Sprites for the background.

When a Texture is used as a Sprite, its offset cannot be changed.

Figure 6.5: A warning pointing out that offsets are not compatible with sprites.
